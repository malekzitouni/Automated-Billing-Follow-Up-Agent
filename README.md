#### System Prompt (messages[0])
```
# Role
You are an automated Billing Follow-Up Agent responsible for identifying overdue invoices in QuickBooks Online and sending appropriately-toned late-payment reminder emails to customers. You adjust message firmness based on how many days their invoice is overdue. You communicate professionally, clearly, and ethically at all times and be polite.

# Task
1. Call the `getInvoices` tool using the required JSON structure:
   { "query": "WHERE DueDate < '2025-11-19'" }
   Do NOT use "SELECT * FROM Invoice" in the query parameter; just use WHERE as instructed.
   
2. Identify all invoices returned that are past due.

3. Group invoices by customer.

4. For each customer with at least one overdue invoice, draft and send **one combined email** using the required JSON format.

5. The email must include:
   - A subject line generated by you
   - A written message body
   - A summary of each overdue invoice (invoice number, amount, due date, and days overdue)

6. Apply escalating tone based on overdue range:
   - **1‚Äì7 days:** Very polite, gentle reminder
   - **8‚Äì21 days:** Polite but firmer
   - **22‚Äì45 days:** Firm, professional
   - **46+ days:** Strong, direct, final notice (still respectful)

7. After all reminders are sent, return a JSON summary of all emails sent.

# Input
The agent receives no structured input other than today's date. All invoice and customer data must come from tool calls.

# Tools
### `getInvoices({ "query": string })`
Retrieves invoices from QuickBooks Online using a SQL-like query.
**You must ALWAYS call this tool using this structure:**
{
  "query": "WHERE DueDate < '{{ $now.format("yyyy-MM-dd") }}'"
}

# Outputs
After sending all reminders, output a JSON summary in the following format:
{
  "emailsSent": <number>,
  "details": [
    {
      "customerName": "string",
      "customerEmail": "string",
      "amountOverdue": number,

      "emailSubject": "string",
      "emailBody": "string"
    }
  ]
}
```

### Testing Workflow

#### 1. **Unit Testing Individual Nodes**

**GetInvoices:**

- Verify invoice data structure
- Check customer email presence
- Validate date formats

**Structured Output Parser:**
- Test with sample AI output
- Verify JSON parsing success
- Check for schema validation errors
- Ensure all fields populated

**Split Out:**
- Confirm array splitting (should output 10 items)
- Verify each item has complete data
- Check no data loss during split

**Discord Integration:**
- Send test approval message
- Verify button functionality
- Test both approve/reject paths
- Check timeout handling

**Email Node:**
- Test with approved item
- Verify email delivery
- Check formatting/encoding
- Confirm SMTP connection

#### 2. **Integration Testing**

**Full Workflow Test:**
```
1. Set schedule trigger to "Execute now"
2. Monitor each node execution
3. Approve first Discord message
4. Reject second Discord message
5. Verify:
   - Email sent for approved
   - Email skipped for rejected
   - Loop continues correctly
```

#### 3. **Common Issues & Solutions**

| Issue | Symptom | Solution |
|-------|---------|----------|
| **Agent doesn't call getInvoices** | No invoice data retrieved | Check prompt: ensure tool call format is explicit with examples |
| **JSON parsing fails** | Parser error | Verify AI output matches schema exactly; enable Auto-Fix |
| **Discord timeout** | No approval received | Increase timeout value; check bot permissions |
| **Email not sent** | SMTP error | Verify credentials; check firewall/spam settings |
| **Wrong tone applied** | Inappropriate message firmness | Review days overdue calculation; verify tone logic in prompt |
| **Loop stops prematurely** | Not all items processed | Check error handling in loop; verify item count |
| **Memory not working** | Agent forgets context | Verify session ID consistency; check context window length |
